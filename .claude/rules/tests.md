# Правила тестирования

## Подход
- TDD: RED → GREEN → REFACTOR
- AAA-паттерн: Arrange / Act / Assert
- Один тест = один сценарий

## Обязательно тестировать
- Все функции из `shared/` (юнит-тесты)
- Граничные значения из `config/balance.json`
- Edge-cases: 0, null, отрицательные значения, максимум

## Запрещено
- Тесты, которые всегда проходят
- Отключение или удаление существующих тестов
- Тестировать реализацию вместо поведения
