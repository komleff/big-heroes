# Продуктовый контекст — Big Heroes

**Обновлён:** 2026-04-02

---

## Описание проекта

Big Heroes — асинхронный hybrid RPG battler на PixiJS v8. Два связанных контура прогрессии: PvE-кампания (рогалик-поход) и PvP-арена (серия боёв). Сингл-клиент, без сервера, с ботами вместо живых игроков.

## Платформа

- Standalone браузер (без серверной части)
- Продуктовый тезис: игра про **подготовку, серию решений и конвертацию подготовки в аренный результат**

## Целевая аудитория

Игроки мобильных RPG-баттлеров (автобатлеры, гарчи). Знакомы с петлёй "копи → бейся → получи награду".

## Главный цикл

```
PvE-поход → добыча (масса, снаряжение, бустеры) → PvP-арена →
→ рейтинг + прочность + масса → аренный сундук → награды → новый поход
```

## Ключевые механики MVP

- **PvE-поход:** 1 глава, 8–10 узлов (бои, магазин, лагерь, события, сундуки, финальный босс)
- **Добыча:** масса, снаряжение (3 слота), бустеры, Gold
- **Рюкзак:** 4 слота; при смерти — теряешь то, что не влезло
- **PvP-арена:** 1 лига, боты; победы/поражения → рейтинг + прочность + масса
- **Аренный сундук:** прогресс только за активные PvP-сессии (не оффлайн)
- **Экономика:** Gold (ремонт, шоп), Gems (рюкзак, хард-действия)
- **Dev-панель:** Gold/Gems/масса/рейтинг/снаряжение/бустеры для QA

## Эмоциональные опоры

- PvE: риск, находки, ощущение похода
- PvP: доминирование, истощение ресурса, напряжение серии
- Meta: рост без жёсткого тупика
