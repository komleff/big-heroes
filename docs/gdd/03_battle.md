# 03 — Боевая система (v4)

## Принцип

Быстрый автобой: применить расходник (опционально) → выбрать команду → анимация 2–3 секунды с числами урона → результат. Игрок влияет через подготовку и выбор команды, не через микроменеджмент.

HP восстанавливается до максимума перед каждым боем (и в PvE, и в PvP).

---

## Параметры героя

| Параметр | Формула | Роль |
|----------|---------|------|
| Здоровье (HP) | = масса | Запас прочности. При 0 — поражение |
| Сила | = масса / 3 + оружие + уровень_масса / 3 | Урон за один удар (до вычета брони) |
| Броня | = щит_броня + расходник_броня | Плоское снижение каждого удара |
| Удача | = аксессуар_удача + расходник_удача | +1% к шансу боя, +2% к шансам умений |

HP = Масса (1:1). Игрок с массой 85 кг имеет 85 HP и силу ~28. Просто, понятно, считается в голове.

## Шесть команд

| Команда | ID | Источник | Ломает | Недоступна если |
|---------|----|----------|--------|----------------|
| Атака | cmd_attack | Всегда | Оружие −1 | — (всегда доступна) |
| Блок | cmd_block | Всегда | Щит −1 | — (всегда доступна) |
| Фортуна | cmd_fortune | Аксессуар | Аксессуар −1 | Аксессуар сломан (durability = 0) |
| Отступление | cmd_retreat | Аксессуар | Аксессуар −1 | Аксессуар сломан |
| Обход | cmd_bypass | Аксессуар | Аксессуар −1 | Аксессуар сломан |
| Полиморф | cmd_polymorph | Аксессуар | Аксессуар −1 | Аксессуар сломан |

Атака и Блок — базовые действия любого бойца. Всегда доступны, даже голыми руками.
Умения аксессуара — требуют прочность > 0. Сломался гримуар — колдовать нечем.

## Формула боя

### Урон за удар

```
Урон = max(1, Сила_атакующего − Броня_защитника)
```

### Вероятность победы (TTK-метод)

```
TTK_игрока = HP_врага / max(1, Сила_игрока − Броня_врага)
TTK_врага  = HP_игрока / max(1, Сила_врага − Броня_игрока)

base_win_chance = TTK_врага / (TTK_игрока + TTK_врага)
luck_bonus = Удача × 0.01
final_win_chance = clamp(base_win_chance + luck_bonus, 0.05, 0.95)
```

При равных параметрах: win_chance = 50%. Кто убивает быстрее — тот чаще побеждает.

### Бросок

```
roll = random()
result = roll < final_win_chance ? ПОБЕДА : ПОРАЖЕНИЕ
```

## Блок — формула выравнивания

Блок подтягивает шанс победы к 50%. Против сильного врага — повышает шансы. Против слабого — понижает. Тактический выбор: видишь на экране, что враг массивнее → жми Блок.

```
attack_win_chance = обычный расчёт через TTK

block_power = 0.3 + броня_щита × 0.05
block_modifier = (0.5 − attack_win_chance) × block_power
block_win_chance = clamp(attack_win_chance + block_modifier, 0.05, 0.95) + luck_bonus
```

| Ситуация | Шанс атакой | Шанс блоком (щит +3) |
|----------|-------------|---------------------|
| Против слабого (base 70%) | 70% | 61% → Атака лучше |
| Против равного (base 50%) | 50% | 50% → Без разницы |
| Против сильного (base 35%) | 35% | 42% → **Блок лучше** |
| Против очень сильного (base 20%) | 20% | 34% → **Блок лучше** |

Без щита: block_power = 0.3 (слабое выравнивание). Со щитом +3: block_power = 0.45. С башенным щитом +7: block_power = 0.65.

Ничья при блоке (только PvE): draw_chance = 15%. Ничья = бой повторяется без потерь, можно выбрать другую команду.

## Шансы умений аксессуара

| Команда | Базовый шанс | Удача | Формула |
|---------|-------------|-------|---------|
| Фортуна | base_win_chance (случ. атака или блок) | +2% за единицу | Двойной бонус удачи |
| Отступление | 70% | +2% за единицу | retreat_chance = 70 + luck × 2 |
| Обход | 60% | +2% за единицу | bypass_chance = 60 + luck × 2 |
| Полиморф | Зависит от гримуара (25–60%) | +2% за единицу | poly_chance = base + luck × 2 |

При провале отступления или обхода: бой происходит как атака, но инициатива у врага (в анимации враг бьёт первым).
При провале полиморфа: бой как обычная атака.

## Анимация боя

### Хронометраж

Сближение (0.3 сек) → обмен ударами 2–3 штуки (1.5–2.0 сек) → финал (0.5 сек). Итого 2.5–3.0 секунды.

### Разброс ударов ±30%

Результат определён до анимации. Анимация генерирует 2–3 удара каждой стороне с разбросом, подогнанным под результат:

```
hit_damage = base_damage × random(0.7, 1.3)
```

- Победа: удары победителя в верхней части разброса, удары проигравшего — в нижней.
- Поражение от слабого: мои удары слабые (×0.7–0.9), его сильные (×1.1–1.3). Игрок видит числа и понимает: «не повезло».
- Победа над сильным (Давид vs Голиаф): мои удары мощные, его слабые. «Повезло!».

Числа над головами: белое = обычный удар, жёлтое = сильный (×1.15+), красное + «МОЩНЫЙ УДАР!» = (×1.25+).

## Различия PvE и PvP

| Аспект | PvE | PvP |
|--------|-----|-----|
| HP перед боем | Максимум (= масса) | Максимум (= масса) |
| Противник | Моб / элита / босс | Бот |
| Победа | +масса, +шанс лута | +рейтинг, +прогресс сундука. Масса без изменений |
| Поражение | Поход завершается досрочно | −10% массы, −рейтинг |
| Ничья (блок) | Бой повторяется | Нет ничьей |
| Отступление | Назад на перекрёсток | Бой не засчитан |
| Обход | Вперёд мимо врага | Не применяется |
| Износ снаряжения | Да (по правилу износа) | Да (по правилу износа) |
