## Роль системы

Единая боевая система для PvE и PvP. Пошаговый бой 1 на 1 (герой против противника). Различается только контекст: тип противника, последствия победы/поражения, награды.

## Ход боя

### Последовательность

1. Ход игрока: расчёт урона героя → применение к HP противника.
2. Проверка: HP противника ≤ 0 → победа, переход к экрану результата.
3. Ход противника: расчёт урона противника → применение к HP героя.
4. Проверка: HP героя ≤ 0 → поражение, переход к экрану результата.
5. Повтор с шага 1.

### Управление

MVP: одна кнопка «Атаковать». Каждое нажатие = один ход игрока + автоматический ответный ход противника.

Post-MVP: добавление навыков (skill_slots), выбор действия (атака / защита / навык).

## Формулы

### Урон за ход

```
raw_damage = base_attack × (1 + mass × mass_damage_coeff) × weapon_modifier
```

### Защита

```
defense = base_defense × (1 + armor_modifier)
```

### Финальный урон

```
actual_damage = max(min_damage, raw_damage − defense)
```

min_damage = 1. Игрок всегда наносит хотя бы 1 урона.

### Критический удар

```
is_crit = random() < crit_chance
final_damage = is_crit ? actual_damage × crit_multiplier : actual_damage
```

### Таблица переменных

| Переменная | Источник | Диапазон тюнинга |
|------------|----------|------------------|
| base_attack (герой) | HeroConfig | 10–15 |
| base_attack (мобы) | MobConfig × chapter_scaling | 5–50 |
| base_defense (герой) | HeroConfig | 5–8 |
| base_defense (мобы) | MobConfig × chapter_scaling | 3–30 |
| mass_damage_coeff | FormulaConfig | 0.005–0.02 |
| weapon_modifier | EquipmentConfig | 1.0–2.5 (по tier) |
| armor_modifier | EquipmentConfig | 0.0–1.5 (по tier) |
| crit_chance | HeroConfig.base_crit + accessory_bonus | 0.05–0.25 |
| crit_multiplier | FormulaConfig.base_crit_mult + accessory_bonus | 1.5–2.5 |
| min_damage | FormulaConfig | 1 |

## HP

### Герой

```
max_hp = base_hp + (mass × hp_per_mass)
```

| Параметр | Диапазон |
|----------|----------|
| base_hp | 80–120 |
| hp_per_mass | 0.1–0.5 |

### Мобы PvE

```
mob_hp = chapter_base_hp × mob_hp_multiplier
```

| Тип моба | mob_hp_multiplier | mob_attack_multiplier |
|----------|-------------------|----------------------|
| combat | 1.0 | 1.0 |
| elite | 1.8–2.5 | 1.3–1.8 |
| boss | 3.0–5.0 | 1.5–2.0 |

### Боты PvP

HP бота определяется его массой (та же формула, что у героя) и снаряжением.

## Контекст боя

BattleSystem получает объект контекста:

| Поле | Тип | Описание |
|------|-----|----------|
| mode | "pve" / "pvp" | Режим боя |
| enemy | EnemyData | Статы противника (HP, атака, защита, снаряжение) |
| rewards | RewardTable / null | Таблица наград за победу. В PvP = null (награды обрабатывает PvpSystem) |

### Различия по режимам

| Аспект | PvE | PvP |
|--------|-----|-----|
| Противник | Моб / элита / босс из PveConfig | Бот из BotFactory |
| HP героя между боями | Сохраняется (не восстанавливается) | Полное восстановление перед каждым боем |
| Награда за победу | Масса + шанс лута из RewardTable | Рейтинг + прогресс сундука (через PvpSystem) |
| Последствие поражения | Поход завершается | Потеря массы + откат рейтинга |
| Износ снаряжения | Нет | durability_cost_per_fight за каждый бой |

## Обратная связь в бою

| Событие | Визуальная обратная связь |
|---------|--------------------------|
| Нанесение урона | Всплывающее число урона над противником, shake-анимация (50ms × 3) |
| Получение урона | Всплывающее число (красное) над героем, shake-анимация |
| Критический удар | Увеличенное число + метка «КРИТ!», усиленный shake |
| HP падает ниже 25% | Полоса HP меняет цвет на красный, пульсация |
| Победа | Баннер «ПОБЕДА!» (ribbon), bounce-анимация героя |
| Поражение | Баннер «ПОРАЖЕНИЕ» (красный ribbon), затемнение |

## Целевые параметры боя

| Метрика | Целевой диапазон | Примечание |
|---------|-----------------|------------|
| Длительность боя (combat) | 4–8 ходов | 15–25 сек |
| Длительность боя (elite) | 8–14 ходов | 25–40 сек |
| Длительность боя (boss) | 12–20 ходов | 40–60 сек |
| Длительность боя (PvP) | 5–10 ходов | 15–30 сек |
| Winrate игрока vs combat | 85–95% | Должен чувствовать контроль |
| Winrate игрока vs elite | 55–70% | Ощутимый риск |
| Winrate игрока vs boss | 40–60% | Кульминация, не гарантирована |
| Winrate игрока vs PvP (по всем ботам) | 60–70% | Предвзятый подбор в пользу игрока |

## Edge cases

- Оба участника умирают в один ход: невозможно, ходы последовательные.
- Противник с 0 defense: actual_damage = raw_damage (min_damage не применяется, если raw_damage > 0).
- Герой без оружия: weapon_modifier = 1.0 (бой голыми руками).
- Герой без брони: armor_modifier = 0.0 (нет защиты).
- Сломанное снаряжение (durability = 0): modifier сбрасывается до дефолта (weapon = 1.0, armor = 0.0, accessory = нет бонуса).
