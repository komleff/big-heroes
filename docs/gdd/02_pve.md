## Роль системы

PvE-кампания — источник ресурсов и подготовка к арене. Контролируемый контур прогрессии с вариативностью и риском. Игрок проходит поход, набирает массу, добывает снаряжение и валюту, затем конвертирует результат в PvP.

## Вход в поход

### Условия запуска

- 1 × campaign_ticket.
- Герой доступен (HP полностью восстанавливается между походами).

### Действия перед стартом

1. Выбор снаряжения из инвентаря в слоты (weapon, armor, accessory). Можно оставить слоты пустыми.
2. Выбор PvE-бустеров (до max_pve_booster_slots штук).
3. Подтверждение входа → списание campaign_ticket → генерация маршрута.

---

## Структура похода

Поход — линейная цепочка узлов с развилками. Игрок не видит всю карту, только текущий шаг и доступные пути.

### Параметры главы

| Параметр | MVP | Диапазон тюнинга |
|----------|-----|------------------|
| total_nodes | 8–10 | 6–14 |
| fork_count | 3–4 | 2–6 |
| paths_per_fork | 2–3 | 2–3 |
| boss_node | последний | всегда последний |

### Визуализация маршрута

```
[Start] ─── [Combat] ─┬─ [Chest]  ─── [Camp] ─┬─ [Elite]  ─── [Shop] ─┬─ [Combat] ─── [BOSS]
                       │                        │                        │
                       └─ [Shop]   ─── ...      └─ [Combat] ─── ...     └─ [Event]  ─── ...
                       │
                       └─ [Event]  ─── ...
```

Каждая вертикальная развилка — выбор игрока. После выбора альтернативные пути исчезают.

## Типы узлов

| Тип | ID | Описание | Частота в генерации |
|-----|----|----------|---------------------|
| Обычный бой | combat | Бой с рядовым мобом. Награда: масса + шанс лута | 35–45% |
| Элитный бой | elite | Усиленный моб. Лучше награды, выше риск | 10–15% |
| Магазин | shop | Покупка предметов за Gold внутри похода | 10–15% |
| Лагерь | camp | Восстановление HP героя | 10–15% |
| Событие | event | Выбор из 2–3 вариантов с разными последствиями | 5–10% |
| Сундук | chest | Лут без боя | 10–15% |
| Босс | boss | Финальный бой главы. Гарантированная награда | 1 на главу (фиксирован) |

### Узел: combat

- Противник: случайный моб из mob_pool главы.
- При победе: масса (mass_per_combat) + шанс лута (combat_loot_chance).
- При поражении: поход завершается → правило выхода.
- HP противника и урон определяются chapter_base_hp × mob_hp_multiplier.

### Узел: elite

- Противник: усиленный моб (HP × 1.8–2.5, атака × 1.3–1.8).
- При победе: масса (mass_per_elite) + гарантированный лут (tier ≥ текущей главы).
- При поражении: поход завершается.
- Иконка узла должна отличаться от combat (красный маркер / череп).

### Узел: shop

- Ассортимент: 3–4 предмета из пула главы + 1 бустер + полный ремонт.
- Цены: base_price × shop_price_multiplier.
- Ремонт в магазине дороже, чем в хабе: repair_cost × shop_repair_multiplier (1.5–2.0).
- Игрок может купить или пропустить.

### Узел: camp

- Восстанавливает HP героя на camp_heal_percent (диапазон: 30–50% от max_hp).
- Не восстанавливает прочность снаряжения (прочность не расходуется в PvE).
- Может предложить событие: «Отдохнуть (+HP)» или «Тренироваться (+масса, −HP)».

### Узел: event

Случайное событие с выбором. Формат: описание ситуации + 2–3 варианта.

Примеры (MVP):

| Событие | Вариант A | Вариант B | Вариант C |
|---------|-----------|-----------|-----------|
| Странник предлагает обмен | Отдать Gold, получить предмет | Отдать предмет из рюкзака, получить массу | Отказаться |
| Ловушка на пути | Обойти (ничего) | Разрядить (шанс: лут или потеря HP) | — |
| Заброшенный алтарь | Молиться (+масса, −Gold) | Осквернить (+Gold, −масса) | Пройти мимо |

Количество событий в пуле MVP: 5–8. Каждое событие имеет ID, текст, варианты и последствия в конфиге.

### Узел: chest

- Лут без боя. Содержимое определяется LootSystem по таблице главы.
- Шанс на предмет tier текущей главы: chest_tier_chance (диапазон: 60–80%).
- Шанс на предмет tier + 1: chest_upgrade_chance (диапазон: 5–15%).

### Узел: boss

- Противник: босс главы (HP × 3.0–5.0, атака × 1.5–2.0).
- Гарантированные награды: масса (mass_per_boss) + 1–2 предмета tier текущей главы + Gold.
- При победе: поход завершается успешно (правило выхода — победа).
- При поражении: поход завершается неудачно (правило выхода — поражение).

## Генерация маршрута

PveSystem генерирует цепочку при старте похода:

1. Взять total_nodes из PveConfig текущей главы.
2. Зафиксировать boss_node в позиции total_nodes (последний).
3. Зафиксировать первый узел как combat (мягкий старт).
4. Распределить fork_count развилок по оставшимся позициям (не в первом, не в последнем). Позиции выбираются равномерно.
5. Для каждой развилки сгенерировать paths_per_fork вариантов. Тип каждого варианта — по взвешенному случайному выбору из таблицы частот.
6. Линейные узлы между развилками — тип по тому же взвешенному выбору.
7. Ограничения: не более 2 боёв подряд; не более 1 магазина на маршрут; не менее 1 лагеря до босса.

## Правила выхода

| Условие | Что происходит с рюкзаком | Что происходит с остальным лутом | Масса |
|---------|--------------------------|----------------------------------|-------|
| Победа над боссом | Содержимое рюкзака → инвентарь | Автопродажа за Gold: sell_price = base_price × sell_multiplier | Вся набранная масса сохраняется |
| Поражение в бою | Содержимое рюкзака → инвентарь | Теряется | retained_mass = gained_mass × defeat_mass_retain_ratio |
| Добровольный выход | Эквивалентно поражению | Теряется | Эквивалентно поражению |

| Параметр | Диапазон тюнинга |
|----------|------------------|
| sell_multiplier | 0.3–0.5 |
| defeat_mass_retain_ratio | 0.3–0.6 |

## Страховки от фрустрации

Экстракшн-механика повышает ценность лута, но и риск churn при непонятных правилах или ощущении несправедливости. Страховки критичны для FTUE.

| Механика | Описание | Условие |
|----------|----------|---------|
| Безопасные походы | Поражение не вызывает потери лута из рюкзака | Первые tutorial_safe_runs (2–3) походов |
| Защищённый слот | Предмет в protected_slots (1 слот) не теряется никогда | Всегда |
| VIP-защита | +1 protected_slot | При активном VIP-статусе |
| Rewarded-страховка | Посмотреть рекламу → сохранить 1 дополнительный предмет при поражении | При поражении (1 раз за поход) |
| Revive (post-MVP) | Rewarded → воскрешение с частью HP, продолжение похода | 1 раз за поход |

## Edge cases

- Рюкзак полон, новый лут: показать экран «Рюкзак полон» → выбрать что выбросить или отказаться от нового предмета.
- Игрок пытается войти в поход без campaign_ticket: показать предложение купить билет за Gems или посмотреть рекламу (post-MVP).
- Игрок закрыл приложение во время похода: при возвращении — восстановить состояние из сохранения (SaveManager). Поход продолжается с текущего узла.
- Все пути на развилке — бои, а HP героя критически низок: это валидная ситуация. Игрок может использовать кнопку «Отступить».
