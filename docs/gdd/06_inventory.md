## Структура инвентаря

| Компонент | ID | Размер (MVP) | Описание |
|-----------|----|-------------|----------|
| Слоты экипировки | equip_slots | 3 | weapon, armor, accessory |

## Экипировка

### Слоты

| Слот | ID | Что влияет |
|------|----|-----------|
| Оружие | weapon | weapon_modifier → формула урона |
| Броня | armor | armor_modifier → формула защиты |
| Аксессуар | accessory | crit_chance / crit_multiplier / special_bonus |

### Операции

| Действие | Правило |
|----------|---------|
| Надеть | Предмет из запаса → слот экипировки. Если слот занят — старый предмет идёт в запас |
| Снять | Предмет из слота → запас |
| Продать | Предмет из запаса → Gold (sell_price = base_price × sell_multiplier) |
| Выбросить (в PvE) | Предмет из рюкзака → удаляется, освобождая слот |

### Параметры предмета

| Поле | Тип | Описание |
|------|-----|----------|
| id | string | Уникальный идентификатор (например wpn_sword_t1) |
| name | string | Отображаемое название |
| slot | "weapon" / "armor" / "accessory" | К какому слоту относится |
| tier | 1–5 | Уровень / редкость |
| modifier | number | Основной модификатор |
| special | string / null | Особое свойство (для аксессуаров) |
| max_durability | number | Максимальная прочность |
| current_durability | number | Текущая прочность |
| base_price | number | Базовая стоимость в Gold |

### Редкость (tier)

| Tier | Название | Цвет маркера | Modifier range | Доступность |
|------|----------|-------------|----------------|-------------|
| 1 | Обычный | #A0AEC0 (серый) | 1.0–1.2 | Глава 1 |
| 2 | Необычный | #48BB78 (зелёный) | 1.2–1.5 | Глава 2 |
| 3 | Редкий | #4299E1 (синий) | 1.5–1.8 | Глава 3 |
| 4 | Эпический | #9F7AEA (фиолетовый) | 1.8–2.2 | Глава 4 |
| 5 | Легендарный | #ED8936 (оранжевый) | 2.2–2.5 | Глава 5 |

## Рюкзак

### Роль

Ограничивает вынос лута из PvE-похода. Создаёт выбор между ценными предметами. Точка монетизации через расширение.

### Правила

1. Действует только во время PvE-похода.
2. Лут, полученный в походе (бои, сундуки, события), попадает в рюкзак.
3. Если рюкзак полон — экран «Рюкзак полон»: выбросить что-то из рюкзака или отказаться от нового предмета.
4. При завершении похода — содержимое рюкзака → запас инвентаря.

### Расширение

| Параметр | Значение |
|----------|----------|
| base_backpack_slots | 4 |
| max_backpack_slots | 8 |
| base_expand_cost | 50–100 Gems |
| expand_cost_multiplier | 1.5–2.0 |

Стоимость N-го расширения:

```
expand_cost(N) = base_expand_cost × expand_cost_multiplier ^ N
```

Где N = текущие слоты − base_backpack_slots (0 для первого расширения).

| Слот | N | Стоимость (при base=80, mult=1.5) |
|------|---|----------------------------------|
| 5-й | 0 | 80 Gems |
| 6-й | 1 | 120 Gems |
| 7-й | 2 | 180 Gems |
| 8-й | 3 | 270 Gems |

### Защищённые слоты

| Условие | Количество protected_slots |
|---------|---------------------------|
| Всегда | 1 (первый слот рюкзака) |
| VIP-статус | +1 |

Предмет в защищённом слоте не теряется при поражении в PvE и не может быть случайно выброшен. Визуально выделен: иконка замка, другой цвет рамки.

## Прочность (durability)

### Роль

Сток, ограничивающий длину PvP-сессии. Создаёт спрос на Gold и мотивацию возвращаться в PvE.

### Правила

- Снаряжение изнашивается **только в PvP** (не в PvE).
- Каждый PvP-бой снижает durability всех надетых предметов.
- При durability ≤ 0 предмет считается сломанным.

### Износ

```
new_durability = current_durability − durability_cost_per_fight
```

| Параметр | Диапазон |
|----------|----------|
| durability_cost_per_fight | 8–15 |

### Сломанный предмет

| Слот | Эффект сломанного предмета |
|------|--------------------------|
| weapon | weapon_modifier = 1.0 (базовый, без бонуса) |
| armor | armor_modifier = 0.0 (нет защиты) |
| accessory | special = null (нет бонуса) |

Предмет остаётся в слоте, но не работает. Визуально: красная пометка «Сломано», потускневшая иконка.

## Ремонт

### Формула

```
repair_cost = (max_durability − current_durability) × tier × repair_cost_per_point
```

| Параметр | Диапазон |
|----------|----------|
| repair_cost_per_point | 0.5–2.0 Gold |

### Пример (при repair_cost_per_point = 1.0)

| Предмет | Tier | Max Dur | Current Dur | Repair Cost |
|---------|------|---------|-------------|-------------|
| Обычный меч | 1 | 100 | 30 | (100−30) × 1 × 1.0 = 70 Gold |
| Кольчуга | 1 | 100 | 0 | (100−0) × 1 × 1.0 = 100 Gold |

### Где доступен ремонт

| Место | Множитель цены |
|-------|---------------|
| Хаб (InventoryScene) | × 1.0 |
| PvE-магазин (ShopScene) | × shop_repair_multiplier (1.5–2.0) |

### Rewarded-скидка

При просмотре рекламы — скидка на ремонт:

```
discounted_cost = repair_cost × (1 − repair_ad_discount)
```

repair_ad_discount = 0.3–0.5 (30–50%).

### Дизайн-ограничения

Ремонт не должен восприниматься как налог:
- Стоимость всегда предсказуема и отображается до подтверждения.
- Gold за 1 PvE-поход покрывает ремонт для 4–6 PvP-боёв с запасом.
- Кнопка «Починить всё» для удобства (суммирует стоимость всех предметов).

## Edge cases

- Запас полон, игрок пытается снять экипировку: показать «Запас полон. Продайте что-нибудь». Не позволять снять.
- Игрок пытается продать надетый предмет: нельзя. Сначала снять.
- Durability ≤ 0 перед PvP-боем: бой разрешён, но предмет не даёт бонуса.
- Все предметы сломаны перед входом на арену: разрешить вход (не жёсткое требование), но предупредить: «Всё снаряжение сломано. Боевые бонусы не действуют».
- Игрок продал все предметы и не имеет снаряжения: разрешить PvE и PvP без снаряжения. Модификаторы = дефолтные.
