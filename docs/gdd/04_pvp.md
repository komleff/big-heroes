## Роль системы

Контур конвертации подготовки в статус (рейтинг), награды (аренный сундук) и ощущение прогресса. Арена — не честный симметричный PvP, а серия выгодных атак, которую игрок инициирует, когда готов.

## Вход на арену

### Условия

- 1 × arena_ticket.
- mass ≥ min_arena_mass текущей лиги.
- Рекомендация: хотя бы 1 предмет экипирован (но не жёсткое требование).

### Что происходит при входе

1. Списание arena_ticket.
2. BotFactory генерирует список из opponent_count ботов.
3. Открывается PvpLobbyScene.

---

## Сессия арены

Арена — не единичный бой, а серия боёв до истощения. Игрок остаётся на арене и выбирает следующего противника, пока не выполнится одно из условий завершения.

### Условия завершения сессии

| Условие | Приоритет |
|---------|-----------|
| mass < min_arena_mass | Принудительное завершение |
| Вся экипировка сломана (durability = 0 у всех надетых предметов) | Принудительное завершение |
| Игрок нажал «Завершить сессию» | Добровольное |

При принудительном завершении — показать сообщение с причиной и итогами сессии.

## Подбор противников

### Генерация ботов

BotFactory создаёт opponent_count ботов при входе на арену. После каждого боя список обновляется (проигранные/побеждённые заменяются новыми).

### Параметры генерации

| Параметр | Значение | Тюнинг |
|----------|----------|--------|
| opponent_count | 5 | 3–7 |
| bot_mass_min_ratio | 0.7 | 0.5–0.8 |
| bot_mass_max_ratio | 1.3 | 1.1–1.5 |
| bot_rating_min_offset | −200 | −300 ... −100 |
| bot_rating_max_offset | +200 | +100 ... +300 |

### Распределение силы

Не равномерное. Цель — дать игроку ощущение «выгодной охоты».

| Категория | Доля | Масса бота vs игрока |
|-----------|------|---------------------|
| Слабые | 60% | ratio < 1.0 |
| Равные | 25% | ratio 0.95–1.05 |
| Сильные | 15% | ratio > 1.05 |

### Логика генерации одного бота

1. bot_mass = player_mass × random(bot_mass_min_ratio, bot_mass_max_ratio) с учётом распределения выше.
2. bot_rating = player_rating + random(bot_rating_min_offset, bot_rating_max_offset).
3. bot_equipment = случайный набор из equipment_pool текущей лиги.
4. bot_name = случайное из предзаданного списка имён.
5. bot_hp = рассчитывается по той же формуле, что и hero_hp (base_hp + bot_mass × hp_per_mass).

### Отображение бота

В PvpLobbyScene каждый бот показывается как карточка (PlayerRow):
- Имя.
- Масса (числом).
- Рейтинг.
- Визуальная индикация силы: зелёный (слабый), жёлтый (равный), красный (сильный).
- Кнопка «Напасть».

## Рейтинг

### Формула (упрощённый Elo)

```
expected = 1 / (1 + 10^((enemy_rating − player_rating) / 400))
rating_change = round(K × (result − expected))
```

| Параметр | Значение |
|----------|----------|
| K | 32 |
| result (победа) | 1 |
| result (поражение) | 0 |

### Стартовый рейтинг

new_player_rating = 1000.

### Ограничения

- rating ≥ 0 (не может быть отрицательным).
- rating_change при победе всегда ≥ min_rating_gain = 1.
- rating_change при поражении всегда ≤ −1 (хотя бы −1).

### Пример

Игрок (1200) атакует бота (1000):
- expected = 1 / (1 + 10^((1000 − 1200) / 400)) = 1 / (1 + 10^(−0.5)) ≈ 0.76
- Победа: rating_change = round(32 × (1 − 0.76)) = round(7.68) = +8
- Поражение: rating_change = round(32 × (0 − 0.76)) = round(−24.32) = −24

Игрок (1000) атакует бота (1200):
- expected ≈ 0.24
- Победа: +24. Поражение: −8.

Вывод: атаковать слабых — безопасно (мало теряешь), но мало получаешь. Атаковать сильных — рискованно, но много получаешь.

## Последствия боёв

### Онлайн-бой (инициированный игроком)

| Результат | Рейтинг | Масса | Прочность | Сундук |
|-----------|---------|-------|-----------|--------|
| Победа | +rating_change | Без изменений | −durability_cost_per_fight | +chest_progress_per_win |
| Поражение | −rating_change | −(mass × pvp_mass_loss_ratio) | −durability_cost_per_fight | +chest_progress_per_loss |

### Параметры

| Параметр | Диапазон |
|----------|----------|
| durability_cost_per_fight | 8–15 |
| pvp_mass_loss_ratio | 0.03–0.07 |
| chest_progress_per_win | 2–3 деления |
| chest_progress_per_loss | 0–1 деление |

### Оффлайн-поражения

Пока игрок не в игре, его слепок может быть атакован (симуляция для ощущения живого мира).

| Аспект | Правило |
|--------|---------|
| Рейтинг | −offline_rating_loss (фиксированное: 5–15) |
| Масса | −(mass × offline_mass_loss_ratio), где offline_mass_loss_ratio = 0.01–0.03 |
| Сундук | Не затрагивается (ни вперёд, ни назад) |
| Прочность | Не затрагивается |
| Максимум за сессию отсутствия | max_offline_attacks = 3–5 |

При входе в игру — показать сводку оффлайн-атак: «Вас атаковали N раз. Рейтинг: −X. Масса: −Y».

## Аренный сундук

### Роль

Основной пейсмейкер прогресса на арене. Мотивирует серию боёв, даёт награды, замыкающие цикл обратно в PvE.

### Механика

- Шкала прогресса на chest_max_progress делений (MVP: 10).
- Заполняется только за онлайн-инициированные бои.
- При заполнении — открывается, выдаёт награды, сбрасывается.

### Прогресс

| Событие | Прогресс |
|---------|----------|
| Победа в онлайн PvP | +chest_progress_per_win (2–3) |
| Поражение в онлайн PvP | +chest_progress_per_loss (0–1) |
| Оффлайн-поражение | 0 |

### Награды сундука

| Тип награды | Вероятность | Назначение |
|-------------|-------------|------------|
| PvE-бустеры | Высокая | Усиление следующего похода |
| Gold | Высокая | Ремонт, покупки |
| Элементы коллекций | Средняя | Перманентная метапрогрессия |
| Осколки героев | Низкая | Разблокировка новых героев (post-MVP) |
| Снаряжение (tier текущей лиги) | Низкая | Прямое усиление |

Конкретный состав награды определяется LootSystem по таблице chest_reward_table текущей лиги.

## Edge cases

- Игрок входит на арену с массой ровно на границе min_arena_mass, проигрывает первый бой и масса падает ниже: сессия завершается принудительно после первого боя. Показать результат.
- Все боты сильнее игрока: невозможно при текущем распределении (60% слабых). Но если rating_range сильно сместился — BotFactory должен гарантировать хотя бы 1 бота слабее игрока.
- Сундук заполняется в середине сессии: показать анимацию открытия сундука → выдать награды → сессия продолжается, шкала сбрасывается.
- Игрок закрыл приложение во время аренной сессии: при возвращении — восстановить состояние. Бой в процессе считается незавершённым (не засчитывается).
