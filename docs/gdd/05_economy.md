## Масса

### Роль

Масса — центральный ресурс экономики. Она одновременно:
- Боевой параметр (увеличивает урон через mass_damage_coeff).
- Входное требование (min_mass для глав и лиг).
- Расходуемый ресурс (теряется при поражениях в PvP).
- Отображаемый статус (видна другим игрокам / ботам в PvP-списке).

### Источники (притоки)

| Источник | Объём (диапазон тюнинга) | Частота |
|----------|-------------------------|---------|
| PvE: combat (победа) | mass_per_combat: 5–20 | Каждый бой |
| PvE: elite (победа) | mass_per_elite: 15–50 | Реже, 1–2 за поход |
| PvE: boss (победа) | mass_per_boss: 50–150 | 1 за поход |
| PvE: event (позитивный исход) | mass_per_event: 5–30 | 0–2 за поход |
| Аренный сундук (награда) | mass_from_chest: 10–50 | При заполнении сундука |

### Стоки (оттоки)

| Сток | Объём | Частота |
|------|-------|---------|
| PvP: поражение (онлайн) | mass × pvp_mass_loss_ratio (0.03–0.07) | Каждое поражение |
| PvP: поражение (оффлайн) | mass × offline_mass_loss_ratio (0.01–0.03) | До max_offline_attacks за сессию |
| PvE: поражение (частичная потеря набранного) | gained_mass × (1 − defeat_mass_retain_ratio) | При поражении в PvE |

### Целевой баланс

За один полный цикл (PvE-поход + PvP-сессия) масса должна расти нетто при winrate ≥ 60% в PvP. Формула целевого прироста:

```
net_mass_per_cycle ≈ pve_mass_gained − pvp_mass_lost
```

Ожидание: net_mass_per_cycle > 0 при нормальной игре, что создаёт ощущение прогресса. При систематических поражениях масса стагнирует — сигнал вернуться в PvE.

### Ограничения

- mass ≥ 0 (не может быть отрицательной).
- Потеря массы округляется вниз (floor).
- При mass = 0: урон = base_attack × weapon_modifier (без бонуса массы).

## Gold (софт-валюта)

### Роль

Основная оборотная валюта. Зарабатывается в PvE, тратится на ремонт и магазин. Должна ощущаться заработанной, а не бесконечной.

### Источники

| Источник | Объём (диапазон) |
|----------|-----------------|
| PvE: combat (победа) | gold_per_combat: 10–30 |
| PvE: elite (победа) | gold_per_elite: 30–80 |
| PvE: boss (победа) | gold_per_boss: 100–300 |
| PvE: chest | gold_per_chest: 20–60 |
| Автопродажа лишнего лута (при победе над боссом) | base_price × sell_multiplier |
| Аренный сундук (награда) | gold_from_chest: 50–200 |

### Стоки

| Сток | Объём |
|------|-------|
| Ремонт снаряжения | repair_cost (см. [06_inventory](06_inventory.md)) |
| Покупки в PvE-магазине | base_price × shop_price_multiplier |
| Покупка бустеров | booster_gold_cost |

### Целевой баланс

Gold, заработанный за 1 PvE-поход, должен покрывать ремонт 1 комплекта снаряжения для 4–6 PvP-боёв с запасом 20–40% на покупки.

```
gold_earned_per_run ≈ repair_cost_full_set × 1.2–1.4
```

Если баланс нарушен в сторону профицита — игрок не чувствует ценности Gold. В сторону дефицита — фрустрация от невозможности играть.

## Gems (хард-валюта)

### Роль

Премиальная валюта. Ощущается ценной, тратится на расширение возможностей и удобства, не на прямую силу.

### Источники

| Источник | Объём |
|----------|-------|
| Покупка за реальные деньги (IAP) | Основной источник |
| Достижения (one-time) | gems_per_achievement: 10–50 |
| Аренный сундук (редко) | gems_from_chest: 5–20 |

### Стоки

| Сток | Объём |
|------|-------|
| Расширение рюкзака | expand_cost (растущая цена, см. [06_inventory](06_inventory.md)) |
| Покупка билетов кампании | campaign_ticket_gem_cost: 20–50 |
| Покупка билетов арены | arena_ticket_gem_cost: 20–50 |
| Мгновенный ремонт (post-MVP) | repair_gem_cost |

### Принцип

Gems покупают удобство и доступ, не прямую силу. Платящий игрок проходит больше циклов в день (больше билетов), выносит больше лута (больше слотов рюкзака), но всё равно должен играть PvE и PvP.

## Билеты

### Кампания (campaign_ticket)

| Аспект | Значение |
|--------|----------|
| Начальный запас | 5 |
| Восстановление | 1 билет каждые campaign_ticket_regen_hours (диапазон: 2–4 часа) |
| Максимум накопления | max_campaign_tickets: 5–8 |
| Покупка за Gems | campaign_ticket_gem_cost |

### Арена (arena_ticket)

| Аспект | Значение |
|--------|----------|
| Начальный запас | 3 |
| Восстановление | 1 билет каждые arena_ticket_regen_hours (диапазон: 3–6 часов) |
| Максимум накопления | max_arena_tickets: 3–5 |
| Покупка за Gems | arena_ticket_gem_cost |

### Дизайн-смысл

Билеты — основной пейсмейкер сессий. Бесплатный игрок может сделать 3–5 PvE-походов и 2–3 PvP-сессии в день. Платящий — больше за счёт покупки билетов.

## Бустеры

### PvE-бустеры

| ID | Название | Эффект | Источник |
|----|----------|--------|----------|
| boost_hp | Зелье здоровья | +hp_boost_percent от max_hp перед походом | Аренный сундук, магазин |
| boost_loot | Талисман удачи | +loot_chance_bonus к шансу лута в боях | Аренный сундук |
| boost_mass | Тоник массы | +mass_gain_bonus к набору массы за поход | Магазин, достижения |

### PvP-бустеры

| ID | Название | Эффект | Источник |
|----|----------|--------|----------|
| boost_attack | Руна атаки | +attack_boost_percent к урону на 1 PvP-сессию | PvE-лут, магазин |
| boost_shield | Руна щита | Первый полученный удар снижен на shield_reduction_percent | PvE-лут |

Количество слотов бустеров: max_pve_booster_slots = 2, max_pvp_booster_slots = 1.

## Служебная dev-панель

Для тестирования баланса необходима dev-панель (см. [09_ui_flows](09_ui_flows.md)), позволяющая:

- Задать gold / gems / mass / rating вручную.
- Выдать любой предмет из EquipmentConfig.
- Выдать / списать билеты.
- Починить всё снаряжение.
- Заполнить / сбросить аренный сундук.
- Сбросить весь прогресс.
