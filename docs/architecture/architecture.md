# Big Heroes Demo — Архитектура и Роадмап

> Источник: Big Heroes — High Concept, MVP Scope и System Design

## Обзор

Клиентская демо-версия игры Big Heroes. Без сервера, с ботами вместо живых игроков. Цель — наглядно показать основной цикл: PvE-поход → подготовка → PvP-арена → награды → новый поход.

Стек: TypeScript + PixiJS.
Сборка: Webpack (client/webpack.config.ts).
Данные: JSON-конфиги, LocalStorage для сохранения.

---

## 1. Ограничения демо

| Аспект | Полный MVP | Демо |
|--------|-----------|------|
| Герои | 1 герой | 1 герой |
| PvE-главы | 1 глава | 1 глава, 8–10 узлов |
| PvP-арена | 1 лига, асинхронные бои | 1 лига, бои против ботов |
| Снаряжение | Полный набор слотов | 3 слота (оружие, броня, аксессуар) |
| Бустеры | PvE + PvP бустеры | 2–3 типа бустеров |
| Экономика | Софт + хард валюта | Только софт-валюта (Gold) |
| Рюкзак | Ограниченный | 4 слота |
| Аренный сундук | Полный прогресс | Упрощённая шкала на 10 делений |
| Dev-панель | Полная | Базовая (Gold, масса, снаряжение) |

### 2.2. Компоненты и ответственности

| Компонент | Ответственность |
|-----------|-----------------|
| Game | Управление жизненным циклом, переключение сцен |
| GameState | Единый источник правды: герой, инвентарь, ресурсы, прогресс |
| EventBus | Связь между системами без прямых зависимостей |
| BattleSystem | Пошаговый расчёт боя: ход игрока, ход противника, применение формул |
| PveSystem | Генерация графа узлов, управление текущим положением в походе |
| PvpSystem | Генерация списка ботов-противников, расчёт рейтинга, прогресс сундука |
| InventorySystem | Слоты экипировки, рюкзак, надевание/снятие предметов |
| LootSystem | Генерация добычи по таблицам из конфига |
| BotFactory | Создание ботов с массой и снаряжением, привязанными к рейтингу игрока |
| SaveManager | Сериализация GameState в LocalStorage, загрузка при старте |
| FormulaEngine | Все числовые расчёты: урон, защита, шанс крита, изменение массы |

### 2.3. Поток данных

```
[Конфиги JSON] → GameState ← SaveManager ↔ LocalStorage
                     ↓
                 EventBus
                 ↓      ↓
           Systems    Scenes (PixiJS)
           ↓              ↓
     FormulaEngine    UI-компоненты
```

Все системы читают и пишут в GameState. Сцены подписываются на EventBus и обновляют отображение. Конфиги неизменяемы в рантайме (кроме dev-панели).

### 2.4. Сцена боя — общая для PvE и PvP

BattleScene получает контекст боя через параметр:

| Поле | Описание |
|------|----------|
| mode | "pve" или "pvp" |
| enemy | данные противника (моб, элита, босс или бот) |
| rewards | таблица наград (для PvE) |

BattleSystem обрабатывает бой пошагово. Каждый ход: расчёт урона атакующего, применение защиты, обновление HP. Бой завершается при HP ≤ 0 одной из сторон.

### 2.5. Генерация похода

PveSystem генерирует линейную цепочку узлов с развилками:

```
[Start] → [Fork: Combat / Chest] → [Fork: Shop / Elite] → ... → [Boss]
```

На каждом шаге игрок выбирает один из 2–3 путей. Каждый путь ведёт к узлу определённого типа. Типы узлов: combat, elite, shop, camp, event, chest. Финальный узел всегда boss.

Для демо достаточно 8–10 узлов в цепочке, из которых 3–4 — развилки.

### 2.6. PvP-боты

BotFactory создаёт ботов по следующей логике:

- Базовая масса бота = масса игрока × коэффициент (0.7–1.3).
- Снаряжение бота — случайный набор из пула, доступного на текущей лиге.
- Имя бота — случайное из предзаданного списка.
- Список из 5 ботов обновляется при входе на арену.

### 2.7. Формулы (ключевые)

| Формула | Описание |
|---------|----------|
| damage = base_attack × (1 + mass × 0.01) × weapon_modifier | Урон за ход |
| defense = base_defense × (1 + armor_modifier) | Снижение входящего урона |
| actual_damage = max(1, damage − defense) | Финальный урон |
| rating_change = K × (result − expected) | Изменение рейтинга (упрощённый Elo) |
| mass_loss_on_defeat = current_mass × 0.05 | Потеря массы при поражении в PvP |
| durability_cost_per_fight = 10 | Износ снаряжения за бой PvP |
| repair_cost = (max_durability − current_durability) × item_tier | Стоимость ремонта |

K = 32 для демо. expected = 1 / (1 + 10^((enemy_rating − player_rating) / 400)).

### 2.8. Dev-панель

Доступна через кнопку в HubScene (или горячую клавишу). Позволяет:

- Задать Gold вручную.
- Задать массу героя.
- Задать рейтинг.
- Выдать любой предмет из EquipmentConfig.
- Починить всё снаряжение.
- Сбросить прогресс.

## 3. Роадмап: 5 спринтов

Каждый спринт рассчитан на одну сессию вайбкодинга с Claude Code (20–40 минут). После каждого спринта — работающая, наглядная версия.

---

### Спринт 1: Скелет и бой

**Цель:** Запускается приложение, на экране происходит пошаговый бой героя с мобом.

**Задачи:**

- S1-01: Инициализация проекта (Vite + TypeScript + PixiJS). P0.
- S1-02: Game.ts — переключение сцен, главный цикл. P0.
- S1-03: GameState.ts — структура состояния героя (HP, масса, атака, защита). P0.
- S1-04: FormulaEngine.ts — расчёт урона и защиты. P0.
- S1-05: BattleSystem.ts — пошаговый бой: ход игрока → ход врага → проверка HP. P0.
- S1-06: BattleScene.ts — визуализация боя: два спрайта, полоски HP, кнопка «Атаковать», лог боя. P0.
- S1-07: Базовые UI-компоненты: Button, ProgressBar. P0.
- S1-08: HeroConfig.ts, простейший конфиг моба. P1.

**Результат:** Открываешь браузер, видишь двух персонажей в стиле Surprise Arena (тёмный фон, неоновые акценты, 3D-кнопки), жмёшь «Атаковать», идёт бой до победы или поражения.

**Зависимости:**

```
S1-01 → S1-02 → S1-03 → S1-04 → S1-05 → S1-06
                                          ↑
                                       S1-07, S1-08
```

---

### Спринт 2: PvE-поход

**Цель:** Игрок проходит маршрут из нескольких узлов, бьёт мобов, собирает лут, доходит до босса.

**Задачи:**

- S2-01: PveConfig.ts — конфиг главы: типы узлов, мобы, награды. P0.
- S2-02: PveSystem.ts — генерация цепочки узлов с развилками. P0.
- S2-03: PveMapScene.ts — экран карты похода: текущий узел, выбор пути. P0.
- S2-04: LootSystem.ts — генерация добычи после боя. P0.
- S2-05: LootScene.ts — экран добычи: показ найденного, кнопка «Взять». P1.
- S2-06: EquipmentConfig.ts — таблица снаряжения (3–5 предметов на слот). P1.
- S2-07: Узел shop: покупка предметов за Gold. P2.
- S2-08: Узел camp: восстановление HP. P2.
- S2-09: Узел chest: лут без боя. P2.
- S2-10: Босс главы — усиленный моб, при победе поход завершается. P1.

**Результат:** Полноценный PvE-поход от начала до босса с боями, добычей и выбором маршрута.

**Зависимости:**

```
S2-01 → S2-02 → S2-03
S2-04 → S2-05
S2-06 (параллельно)
S2-07, S2-08, S2-09 (независимы, после S2-03)
S2-10 (после S2-03 + S2-04)
```

---

### Спринт 3: Инвентарь, рюкзак, масса

**Цель:** Работает экипировка, рюкзак ограничивает вынос лута, масса влияет на урон.

**Задачи:**

- S3-01: InventorySystem.ts — слоты экипировки (оружие, броня, аксессуар), надевание/снятие. P0.
- S3-02: InventoryScene.ts — экран инвентаря: слоты, карточки предметов, статы. P0.
- S3-03: Рюкзак — 4 слота для выноса лута из похода. P0.
- S3-04: Правило выноса — при поражении теряется лишний лут; при победе лишнее продаётся. P1.
- S3-05: Масса — отображение, начисление в PvE, влияние на формулу урона. P0.
- S3-06: Прочность снаряжения — отображение, износ пока только подготовлен. P2.
- S3-07: HubScene.ts — хаб между PvE и PvP: инвентарь, кнопки «Поход» и «Арена». P0.

**Результат:** Игрок экипируется, идёт в поход, собирает добычу в рюкзак, возвращается в хаб с лутом. Масса растёт, статы меняются.

---

### Спринт 4: PvP-арена

**Цель:** Работает PvP-арена с ботами, рейтинг, износ, аренный сундук.

**Задачи:**

- S4-01: BotFactory.ts — генерация 5 ботов по рейтингу игрока. P0.
- S4-02: PvpSystem.ts — аренная сессия: выбор цели, бой, обновление рейтинга. P0.
- S4-03: PvpLobbyScene.ts — список ботов с массой и рейтингом, кнопка «Атаковать». P0.
- S4-04: PvpResultScene.ts — результат боя, изменение рейтинга, износ. P0.
- S4-05: PvpConfig.ts — формула рейтинга, параметры K, масса ботов. P1.
- S4-06: Износ снаряжения в PvP — durability_cost_per_fight за каждый бой. P1.
- S4-07: Потеря массы при поражении — mass_loss_on_defeat. P1.
- S4-08: Аренный сундук — шкала прогресса, награда при заполнении. P1.
- S4-09: Условие завершения PvP-сессии — масса слишком низкая или прочность кончилась. P2.

**Результат:** Полный цикл арены: выбор бота → бой → результат → обновление рейтинга → износ → сундук. Цикл PvE → PvP замкнут.

---

### Спринт 5: Ремонт, экономика, dev-панель, полировка

**Цель:** Замкнутая экономика, ремонт, сохранение, dev-панель. Демо готово к показу.

**Задачи:**

- S5-01: RepairSystem.ts — ремонт снаряжения за Gold. P0.
- S5-02: EconomyConfig.ts — цены, награды, баланс Gold. P0.
- S5-03: SaveManager.ts — сохранение в LocalStorage, загрузка при старте. P0.
- S5-04: DevPanelScene.ts — ручное управление Gold, массой, рейтингом, снаряжением. P1.
- S5-05: ResourceBar.ts — постоянное отображение Gold и массы. P1.
- S5-06: MainMenuScene.ts — стартовый экран с кнопками «Играть», «Продолжить». P2.
- S5-07: Визуальная полировка: анимации ударов, переходы между сценами. P3.
- S5-08: Звуковые эффекты (опционально, если время позволяет). P4.

**Результат:** Полностью играбельная демо-версия. Замкнутый цикл PvE → экипировка → PvP → ремонт → новый PvE. Сохранение между сессиями. Dev-панель для тестирования.

## 4. Приоритеты для первой сессии (2 часа)

Если цель — за 2 часа получить работающий прототип, рекомендуемый фокус:

**Обязательно:** Спринт 1 + Спринт 2 (ядро).
Это даст: запуск, бой, PvE-поход с картой, лут, босс.

**Желательно:** Спринт 3 (частично: инвентарь + хаб).
Это добавит: экипировку и точку возврата между контурами.

**Если останется время:** Спринт 4 (PvP-арена хотя бы на уровне «выбрал бота → бой → рейтинг»).

Спринт 5 — следующая сессия.

## 5. Инструкция для Claude Code

При работе с Claude Code рекомендуется давать задачи поспринтово. Пример промпта для начала:

> Создай проект TypeScript + PixiJS + Webpack. Структура папок — как в архитектуре (src/core, src/systems, src/scenes, src/ui, src/config, src/utils). Визуальный стиль: тёмно-синий фон (#1C2340), портретная ориентация 390×844, скруглённые карточки (#3D4670, borderRadius 12), 3D-кнопки с градиентом (зелёные #5EC040 для основного действия), крупные числа массы голубым (#4AE0E0), текстовые тени, шрифт Nunito. Реализуй спринт 1: ThemeConfig с токенами цветов, Game.ts для переключения сцен, GameState с параметрами героя, FormulaEngine с формулами урона, BattleSystem для пошагового боя, BattleScene в стиле Surprise Arena — два «яйца» (скруглённые квадраты с аватарами), полоски HP, кнопка «Атаковать» (зелёная 3D), информация о массе и рейтинге. При нажатии — пошаговый бой до конца. Экран результата с баннером «ПОБЕДА!» или «ПОРАЖЕНИЕ» в стиле ленты.

После каждого спринта — проверяй в браузере, фиксируй проблемы, переходи к следующему.

## 6. Визуальный стиль

Визуальный стиль основан на макетах Surprise Arena (см. [style-guide](../design/style-guide.md)).

Ключевые принципы:
- Портретная ориентация, 390×844 (мобильный формат).
- Тёмно-синий фон (#1C2340), яркие неоновые акценты.
- 3D-кнопки с нижней тенью и градиентом.
- Скруглённые карточки (12px) для списков и предметов.
- Крупные числа массы голубым цветом (accent_cyan).
- Зелёные кнопки для основных действий, розовые — для вторичных.
- Баннеры-ленты для заголовков результатов (Победа / Поражение).
- Bounce-анимации при получении наград.

ThemeConfig.ts содержит все токены цветов, размеров и шрифтов. Все сцены читают стиль из ThemeConfig, не хардкодят значения.

## 7. Глоссарий

| Термин | Определение |
|--------|-------------|
| Масса | Основной боевой ресурс героя, растёт в PvE, теряется при поражении в PvP |
| Прочность | Ресурс снаряжения, снижается за каждый PvP-бой |
| Аренный сундук | Шкала прогресса в PvP, заполняется за активные бои |
| Рюкзак | Ограниченное число слотов для выноса лута из PvE-похода |
| Узел | Точка на маршруте похода: бой, сундук, магазин, лагерь, событие |
| Бот | Автоматически сгенерированный противник для PvP-арены |
| Gold | Софт-валюта для ремонта и покупок |
| Развилка | Точка на маршруте, где игрок выбирает один из 2–3 путей |
