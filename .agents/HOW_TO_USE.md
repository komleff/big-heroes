# Как пользоваться пайплайном

**Для:** оператора (менеджера), который не читает код
**Обновлён:** 2026-04-02

---

## 1. Быстрый старт

### Что нужно

- Терминал или VS Code с Claude Code
- Открыта папка проекта `big-heroes`
- Доступ к GitHub (для PR и merge)

### Первый запуск

1. Открой Claude Code в папке проекта
2. Напиши в чат:

   ```
   Ты PM. Прочитай .agents/AGENT_ROLES.md секция "0. Project Manager".
   Задача: покажи текущее состояние проекта
   ```

3. Claude прочитает Memory Bank и расскажет, что сейчас происходит

---

## 2. Как запустить спринт

### Активация PM

```
Ты PM. Прочитай .agents/AGENT_ROLES.md секция "0. Project Manager".
Задача: спланируй Sprint N — [название/цель спринта]
```

PM сделает:

- Прочитает текущее состояние (`status.md`)
- Создаст задачи в Beads (`bd create`)
- Определит приоритеты и зависимости
- Предложит план спринта

### После планирования

Скажи PM:

```
Начинай реализацию. Делегируй задачи Developer-субагентам.
```

---

## 3. Как использовать /sprint-pr-cycle

### Когда использовать

После того как Developer завершил кодирование спринта.

### Как запустить

Напиши в чат:

```
/sprint-pr-cycle
```

Скилл проведёт через все шаги: сборка → тесты → PR → ревью → отчёт.

---

## 4. Как принимать решение о merge

Смотри только на вердикты в PR:

- `APPROVED` от Claude + `APPROVED` от внешней модели → можно мержить
- `CHANGES_REQUESTED` → PM сам исправит через Developer, скажи «устрани замечания»

Ты **не читаешь код** — только вердикты.

## 5. Частые команды оператора

| Команда | Что скажи |
|---------|----------|
| Начать спринт | "Ты PM. Прочитай .agents/AGENT_ROLES.md секция \"0. Project Manager\". Задача: спланируй Sprint N — [цель]" |
| Запустить реализацию | "Начинай реализацию. Делегируй задачи Developer-субагентам." |
| Запустить ревью-цикл | "/sprint-pr-cycle" |
| Ревью PR вручную | "Ты Reviewer. Прочитай .agents/AGENT_ROLES.md секция \"3. Reviewer\". Задача: проверь PR #N. Уровень ревью: [Light / Standard / Critical]." |
| Исправить замечания | "Устрани все CHANGES_REQUESTED" |
| Показать статус | "Покажи текущее состояние" |
